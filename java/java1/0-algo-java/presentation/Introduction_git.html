1 Gestionnaire de version décentralisé

1.1 Gestion de version

  • historique : versions identifiée et décrites (commentaire)
  • changement d'état atomique : commit des changements d'un ensemble de fichiers.

1.2 Décentralisé

  • Chaque poste de travail possède l'ensemble de l'historique des

versions (un dépôt ou repository).

  • On synchronise les différents dépôts locaux des membres de l'équipe avec un dépôt de référence distant (remote).

http://www.ndpsoftware.com/git-cheatsheet.html

  • Système de fichiers
  • Index

2.1 Création d'un dépôt

  • git init : création locale
  • git clone : création à partir d'un dépôt distant

2.2 Informations sur le dépôt

  • git status
  • git log

2.3 Création d'une nouvelle version

  • git add fichier1
  • git add fichier2
  • …
  • git commit -m "message de commit"

2.4 Changement de version

  • git reset

HEAD, HEAD^, HEAD^^, identifiant

2.5 Mise jour à partir du dépôt distant

  • git pull
  • git pull –rebase

2.6 Mise à jour du dépôt distant

  • git push

  • on garde l'historique de l'ensemble des sous-répertoires du projet (par exemple pour avoir les fichiers de config)
  • on exclue (cf. .gitignore) :
